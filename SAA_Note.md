ログファイルの処理とは？
**ステムやアプリケーションが日々出力する**「操作履歴やイベントの記録（ログ）」を収集・整理・分析する一連の流れのこと**


Amazon Athena
**S3にあるデータをインフラ管理が面倒でしたくない、サクッとSQLで分析したいならAthena**


Amazon Redshift
**構造化されたデータを高速にBIツールなどで分析したいならRedshift**


AWS Glue
**サーバーレスでデータ変換を走らせたいならGlue**


Amazon Kinesis
**曲がれてくるデータを「秒単位」で即座に処理するならKinesis**



Hadoopとは？
**膨大なデータを、大量のコンピューターで分散して処理するためのサービス**


Amazon Athena
**サーバーレスで自動スケーリングで使いやすいS3内のデータをSQLを使用する分析ツール**


ELB(ロードバランサー)
**トラフィックを2つ以上のEC2インスタンスに分散する**


DynamoDB
**NOSQLでリアルタイムで高速なデータの書き・読みが可能**


Aurora
**高いパフォーマンスと可用性(落ちない)を持ったMySQL/PostgreSQL互換のRDBサービス**


Aurora serverless
**トラフィックの急増が予測困難なサービスに最適で、使用状況に応じて自動で拡張と縮小を行い、コストパフォーマンスが高い**


Service Control Policy
**組織内のすべてのアカウントに対し一元的にアクセス生業を適用できるサービス**


AWS Config
**検知と通知を行う監視ツール**


AWS Route 53
**可用性が高いスケーラブルなドメインネームシステム(DNS)**


AWS EMR
**ビックデータ処理および分析を低コストで行える**


MFA Delete
